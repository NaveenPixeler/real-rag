# Contributing to Real RAG

First off, thank you for considering contributing to Real RAG! It's people like you that make this tool better for everyone.

## Development Workflow

1. **Fork the repository** and clone it locally.
2. **Create a branch** for your edits. We follow the standard industry prefix naming convention:
   - `feature/` for new features (e.g., `feature/implement-chunking`)
   - `bugfix/` for bug fixes
   - `docs/` for documentation changes
   - `refactor/` for code refactoring
   - `chore/` for maintenance tasks

3. **Make your changes**. Ensure you write clean code and include tests if applicable.

4. **Commit your changes**. We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
   - `feat: add PDF chunking logic`
   - `fix: resolve 400 error on upload`
   - `style: update chat input pill shape`
   - `docs: update setup instructions`

5. **Push to your fork** and **Submit a Pull Request** against the `main` branch.

## Code Style & Standards

- **Frontend**: We use ESLint and Prettier. Ensure you run `npm run lint` before committing.
- **Backend**: NestJS conventions apply. Formatting is handled via Prettier (`npm run format`).

## Submitting a Pull Request (PR)

- Fill out the PR template completely.
- Describe *why* you are making the changes, not just *what* you changed.
- Link to any relevant issues.
- Request reviews from repository maintainers.

We look forward to reviewing your contributions!
